# スプレッドシート列設計

## シートの目的

Googleフォームから受け付けた問い合わせを一覧管理し、LINE通知、対応状況、再通知、エラー内容を追跡するための管理シートです。

Phase 1では、Googleフォーム回答を `問い合わせ管理` シートへ転記し、LINE通知結果と未対応問い合わせの再通知結果を記録します。

## シート名

標準シート名は `問い合わせ管理` です。

`initializeInquirySheet()` を実行すると、同名シートが存在しない場合は自動作成します。

Googleフォームの回答先として作成されるフォーム回答シートとは別に、運用管理用として `問い合わせ管理` シートを使います。

## 標準列

| 列名 | 内容 | 備考 |
| --- | --- | --- |
| 受付ID | 問い合わせごとの一意な管理ID | 将来的に自動採番予定 |
| 受付日時 | 問い合わせを受け付けた日時 | Googleフォーム回答日時を想定 |
| 氏名 | 問い合わせ者の氏名 | フォーム入力値 |
| メールアドレス | 問い合わせ者のメールアドレス | フォーム入力値 |
| 問い合わせ内容 | 問い合わせ本文 | フォーム入力値 |
| AI要約 | 問い合わせ内容の要約 | OpenAI APIで自動生成 |
| 分類 | 問い合わせ種別 | OpenAI APIで自動生成 |
| 緊急度 | 対応優先度 | OpenAI APIで自動生成。高、中、低 |
| 確認事項 | 担当者が次に確認すべき項目 | OpenAI APIで自動生成 |
| 担当者 | 対応する担当者 | 手動入力または将来の自動割り当て |
| 対応ステータス | 対応状況 | 下記の想定値を使用 |
| LINE通知ステータス | LINE通知の状態 | 下記の想定値を使用 |
| LINE通知日時 | 最初にLINE通知した日時 | 自動記録予定 |
| 再通知回数 | 未対応時の再通知回数 | 再通知成功時に自動更新 |
| 最終再通知日時 | 最後に再通知した日時 | 再通知成功時に自動更新 |
| 対応期限 | 対応すべき期限 | 緊急度や運用ルールから設定予定 |
| エラー内容 | 処理中に発生したエラー内容 | 通知失敗などを記録 |

## Googleフォーム項目

`setupInquiryFormIntegration()` で作成するGoogleフォームには、以下の項目を用意します。

| フォーム項目 | 管理シートの対応列 | 必須 |
| --- | --- | --- |
| 氏名 | 氏名 | 必須 |
| メールアドレス | メールアドレス | 必須 |
| 問い合わせ内容 | 問い合わせ内容 | 必須 |

フォーム送信時には、受付日時、受付ID、対応ステータス、LINE通知ステータスなどをApps Script側で補完します。

## ヘッダー初期化ルール

- 標準ヘッダーは1行目に作成します。
- 既存の1行目が空の場合は、標準列を左から順番に作成します。
- 既存のヘッダーがある場合は、既存列を削除・上書きしません。
- 不足している標準列だけを右側へ追加します。
- 1行目は固定表示し、ヘッダーとして見やすいように太字と背景色を設定します。
- 本番データや既存行は削除しません。

## 対応ステータス

- 未対応
- 対応中
- 対応済み
- 保留

## LINE通知ステータス

- 未通知
- 通知済み
- 再通知済み
- 通知エラー

## 未対応アラート条件

`sendUnhandledInquiryAlerts()` は、以下の条件をすべて満たす行だけを再通知対象にします。

- `対応ステータス` が `未対応`
- `受付日時` から24時間以上経過している
- `再通知回数` が3回未満
- `最終再通知日時` が空、または前回再通知から24時間以上経過している

再通知成功時は、以下の列を更新します。

- `LINE通知ステータス`: `再通知済み`
- `再通知回数`: 現在値 + 1
- `最終再通知日時`: 実行日時
- `エラー内容`: 空欄

## 未対応アラートのテスト行

`createTestUnhandledInquiryAlertRow()` を実行すると、再通知条件を満たすテスト用の未対応行を作成します。

- `受付ID` は `TEST-ALERT-` で始まります。
- `受付日時` は実行時点から25時間前に設定します。
- `対応ステータス` は `未対応` に設定します。
- `再通知回数` は `0` に設定します。
- この関数だけではLINE通知は送信しません。

## Phase 1での扱い

Phase 1では、シート作成、ヘッダー初期化、Googleフォーム回答の転記、OpenAI APIによるAI要約・分類・緊急度・確認事項の生成、LINE通知ステータス更新、未対応問い合わせの再通知までを行います。
