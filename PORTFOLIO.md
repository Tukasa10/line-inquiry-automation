# ポートフォリオ説明

## 制作物名

LINE問い合わせ通知・未対応アラート自動化システム

## 制作目的

Googleフォームに届いた問い合わせを自動で整理し、担当者がすぐ確認できるようにLINEへ通知する業務自動化ツールです。

問い合わせ内容をAIで要約・分類し、未対応のまま残った問い合わせは再通知することで、確認漏れや対応遅れを減らすことを目的にしています。

## 想定する利用者

- 小規模事業者
- 個人事業主
- 店舗運営者
- 予約、相談、見積もり問い合わせをGoogleフォームで受けている事業者
- 問い合わせ対応を少人数で管理しているチーム

## 対応できる業務課題

- フォーム問い合わせに気づくのが遅れる
- 問い合わせ内容の確認に時間がかかる
- 緊急度の高い問い合わせを優先しにくい
- 未対応の問い合わせが埋もれる
- LINE通知後の対応状況を追跡しづらい
- 再通知や対応漏れ防止を手作業で行っている

## 使用技術

- Google Apps Script
- Googleフォーム
- Googleスプレッドシート
- LINE Messaging API
- OpenAI API
- Script Properties
- clasp

## 実装機能

- Googleフォーム作成
- フォーム回答のスプレッドシート記録
- `問い合わせ管理` シートの自動初期化
- OpenAI APIによる問い合わせ分析
  - AI要約
  - 分類
  - 緊急度
  - 確認事項
- AI結果入りLINE通知
- LINE通知ステータス管理
- 未対応問い合わせのLINE再通知
- 再通知回数の記録
- 最終再通知日時の記録
- 未対応アラート用トリガー作成
- テスト用関数による動作確認
- claspによるApps Script反映

## 処理フロー

1. Googleフォームから問い合わせが送信されます。
2. Apps Scriptがフォーム回答を取得します。
3. 回答内容を `問い合わせ管理` シートへ記録します。
4. OpenAI APIで問い合わせ内容を要約、分類、緊急度判定します。
5. AI結果と問い合わせ内容をLINEへ通知します。
6. 通知結果をスプレッドシートへ記録します。
7. 定期実行で未対応問い合わせを確認します。
8. 一定時間以上未対応の問い合わせをLINEへ再通知します。
9. 再通知回数と最終再通知日時を記録します。

## 工夫した点

- APIキーやLINEトークンをScript Propertiesで管理し、コードに直書きしない構成にしました。
- スプレッドシートの列はヘッダー名ベースで取得し、列順変更にある程度強い設計にしました。
- LINE通知ステータスを記録し、通知済み、再通知済み、通知エラーを後から追跡できるようにしました。
- 再通知回数を管理し、同じ問い合わせを無制限に通知しない設計にしました。
- 未対応アラートにより、対応漏れを防げるようにしました。
- claspを使い、ローカルのApps ScriptコードをGoogle側へ反映しやすくしました。
- OpenAI分析だけ、フォーム送信相当、未対応アラートなど、段階別に確認できるテスト用関数を用意しました。
- 本番データを削除せず、不足ヘッダーだけを追加する初期化処理にしました。

## 安全設計

- `LINE_CHANNEL_ACCESS_TOKEN`、`LINE_TO_ID`、`OPENAI_API_KEY` はScript Propertiesで管理します。
- 秘密情報の実値はコードやREADMEに記載しません。
- `.clasp.json` と `.clasprc.json` はGit管理対象外にします。
- LINE通知失敗時はレスポンスコードと本文をログに残します。
- 未対応アラートは再通知回数の上限を持ち、過剰通知を防ぎます。
- テスト用の未対応問い合わせは `TEST-ALERT-` で始まる受付IDにし、通常データと見分けられるようにしています。
- Webhook / `doPost` やLINEから直接ステータス変更する機能は、現時点では実装していません。
